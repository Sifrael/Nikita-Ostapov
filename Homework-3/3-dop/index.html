Алгоритм распознования цветов

При распозновании значения цвета элемента браузер использует шестнадцатеричную система счисления. В качестве цифр этой системы счисления обычно используются цифры от 0 до 9 и латинские буквы от A до F. Буквы A, B, C, D, E, F имеют значения 1010 , 1110, 1210, 1310, 1410, 1510 соответственно. Некорректные символы распознаются как 0. Так что, например, значения #F0F0F0, F0F0F0, F0F0F, #FxFxFx и FxFxFx – одно и то же. Для распозновая цветов используется алгоритм описанный ниже.

Алгоритм цвета #BruceWillis

  1. Заменяем все неправильные шестнадцатеричные символы нулями 0, BruceWillis превращается в b00ce000000
  2. “Добиваем” число нулями до количества знаков, делимого на 3 (с 11 до 12), получаем b00ce0000000
  3. Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB: RGB (b00c, e000, 0000)
  4. Обрезаем каждый из аргументов справа(если возможно), оставляя 2 символа, и получаем наш результат RGB (b0, e0, 00) = #B0E000 = RGB(176, 224, 0).

Результат работы алгортима используется в качестве фона для данного блока.

Алгоритм цвета #CANDY

  1. Заменяем все неправильные шестнадцатеричные символы нулями 0, CANDY превращается в ca0d0
  2. “Добиваем” число нулями до количества знаков, делимого на 3 (с 5 до 6), получаем ca0d00
  3. Разбиваем на три группы, каждая из которых отвечает за одну компоненту RGB: RGB (ca, 0d, 00)
  4. Обрезаем каждый из аргументов справа(если возможно), оставляя 2 символа, и получаем наш результат RGB (ca, 0d, 00) = #CA0D00 = RGB(202, 13, 0).

Результат работы алгортима используется в качестве фона для данного блока.

Безопасная палитра цветов

Ограничение в 216 цветов, которые гарантированно будут отображены браузером без искажений, исходит из того, что некоторые компьютеры способны отображать только 256 цветов. Часть из них используется для системной палитры (к примеру для окон). Так что для Web'a остается только 216. Сейчас использование безопасной палитры не является необходимым условием для создания сайта. Хотя сайты, заботящиеся обо всех своих посетителях, или, по крайней мере, о подавляющем их большинстве, придерживаются безопасных цветов.

Когда компьютер не способен отобразить какой-либо цвет, он пытается имитировать его с помощью смешения. Т.е. он смешивает доступные ему цвета в нужной пропорции. Однако результат будет далек от идеального, и при близком рассмотрении такая картинка выглядит зернистой. Особенно это заметно на цветовых растяжках. Так что при использовании смешения градиенты противопоказаны. Если вы используете только цвета из безопасной палитры, то они не будут подвергнуты смешению, и вы можете быть уверены, что сделанная вами страница, на чужом мониторе будет смотреться, как на вашем (если не принимать во внимание особенности цветопередачи разных мониторов).
